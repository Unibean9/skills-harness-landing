# Product

## Register

brand

## Users

Frontend developers, design engineers, AI-coding-agent users, and curious builders who want a clearer way to produce high-quality UI with agent workflows. They may be discovering fk-skills for the first time and need to quickly understand what it is, how to install it, and why it is useful.

## Product Purpose

This landing page introduces the fk-skills package as a design-focused skill system for AI coding agents. It should make the install path obvious, explain the benefits, show the harness and command model, and demonstrate the strongest design rules and features in a memorable way.

## Brand Personality

Precise, kinetic, and premium. The page should feel like a frontend craft showcase: technically credible, visually ambitious, and exciting enough that visitors want to try the skill immediately.

## Anti-references

Avoid generic SaaS card grids, beige AI landing pages, vague "modern clean" styling, template hero metrics, repeated tiny uppercase section labels, and decorative animation that does not explain the skill. The page should not feel like a docs index or a plain package README.

## Design Principles

- Make installation unmistakable in the hero.
- Demonstrate the skill's design taste through the page itself.
- Treat commands, harnesses, rules, and features as tangible product capabilities.
- Use motion to clarify hierarchy and create delight, not to hide thin content.
- Keep copy direct, specific, and useful for frontend builders.

## Accessibility & Inclusion

Target WCAG AA contrast, semantic structure, keyboard-friendly interactions, and responsive behavior across mobile and desktop. Provide reduced-motion alternatives for GSAP or scroll-linked animation.
